# TrainConsistManagementApp

It is a modular Java console application for managing train consists \

---

# Use Cases

## UC-01: Initialize Train and Display Consist Summary

Initialize the application and display the initial state of the train.
Train is empty and does not support dynamic bogie operations.

---

## UC-02: Add Passenger Bogies (ArrayList Operations)

Add and remove passenger bogies dynamically using ArrayList.
Duplicate bogies are allowed.

---

## UC-03: Track Unique Bogie IDs (HashSet)

Ensure uniqueness of bogie IDs using HashSet.
Order of bogies is not preserved.

---

## UC-04: Maintain Ordered Bogie IDs (TreeSet / SortedSet)

Maintain a sorted and predictable order of bogies.
Less flexible for insertion and removal.

---

## UC-05: LinkedList for Train Chaining

Model train chaining with efficient insertion and removal using LinkedList.

---

## UC-06: Validate Train Consist

Validate rules like minimum bogies and capacity limits.

---

## UC-07: Safety Compliance Check

Check safety rules such as brake ratio and weight limits.

---

## UC-08: Group Bogies by Type

Group bogies into categories like passenger and freight.

---

## UC-09: Count Bogies by Attribute

Count bogies based on type, capacity, or other attributes.

---

## UC-10: Search Bogies

Search bogies by ID, type, or capacity.

---

## UC-11: Sort Bogies

Sort bogies based on ID, type, or capacity.

---

## UC-12: Performance Benchmarking

Compare loop vs stream performance using System.nanoTime().

---

## UC-13: Exception Handling

Handle invalid inputs and missing bogies using custom exceptions.

Perform bulk actions like delete, validation, or export.

Filter bogies using multiple attributes.

Generate reports such as total bogies and compliance status.

Log errors and implement recovery strategies.
